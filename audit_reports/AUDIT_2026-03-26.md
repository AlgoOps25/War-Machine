# War Machine Audit & Fix Plan - March 26, 2026

## CRITICAL ISSUES (Fix Immediately — Direct Win Rate Impact)

### 1. ML Boost Is Dead Code (sniper_pipeline.py)
**Problem:** compute_confidence() returns a static value. ml_boost = 0.0 is hardcoded. MLConfidenceBooster exists but is never called.
**Fix:** 
- Import MLConfidenceBooster and AILearningEngine.
- Wire ml_boost using predict_confidence_adjustment(ml_features).
- Wire ticker_multiplier using get_ticker_confidence_multiplier(ticker).

### 2. RVOL Ceiling Penalty Double-Enforced
**Problem:** RVOL >= 3.0 is blocked at both pipeline gate and scorecard deduction. Scorecard stats never show the deduction because pipeline returns False early.
**Fix:** Remove the early-return RVOL_CEILING block from sniper_pipeline.py. Let the scorecard enforce it via the -20 pts deduction for better observability.

### 3. _resample_bars() Duplicated
**Problem:** Identical functions in sniper.py and sniper_pipeline.py lead to logic divergence (MTF mismatch bug).
**Fix:** Move to utils/bar_utils.py and import from both callers.

### 4. Grade Weight Flattening vs Confidence Formula
**Problem:** Scorecard uses flattened weights (15/13/11), but compute_confidence uses wider gaps (0.90/0.85/0.80).
**Fix:** Add RVOL-conditional confidence floor in compute_confidence() to cap A+ at B+ level when RVOL < 1.2x.
