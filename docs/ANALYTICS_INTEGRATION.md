# War Machine Analytics Integration Guide\n\nThis guide shows how to integrate the signal analytics system into your main War Machine trading bot.\n\n## Files Created\n\n1. **`src/analytics/signal_analytics.py`** - Core tracking + deduplication (11.2 KB)\n2. **`src/learning/ml_feedback_loop.py`** - ML training pipeline (6.4 KB)\n3. **`src/reporting/performance_reporter.py`** - Daily stats generator (8.2 KB)\n4. **`database/signal_outcomes_schema.sql`** - Database schema (4.0 KB)\n\n---\n\n## Step 1: Database Setup\n\n### Create Tables in PostgreSQL\n\n```bash\n# Connect to Railway PostgreSQL\npsql $DATABASE_URL -f database/signal_outcomes_schema.sql\n```\n\n**What this does:**\n- Creates 3 tables: `signal_outcomes`, `pattern_performance`, `ml_training_data`\n- Adds indexes for fast queries\n- Inserts your NVDA 60% winner as bootstrap training data\n\n---\n\n## Step 2: Update main.py\n\n### 2.1 Add Imports (Top of File)\n\n```python\nfrom src.analytics.signal_analytics import SignalAnalytics\nfrom src.learning.ml_feedback_loop import MLFeedbackLoop\nfrom src.reporting.performance_reporter import PerformanceReporter\n```\n\n### 2.2 Initialize After DB Connection\n\n```python\n# After establishing db_connection\nanalytics = SignalAnalytics(db_connection)\nml_loop = MLFeedbackLoop(db_connection)\nreporter = PerformanceReporter(\n    db_connection,\n    discord_webhook_url=os.getenv('DISCORD_WEBHOOK_URL')  # Add this env var\n)\n\n# Try to load existing ML model\nml_loop.load_model('models/war_machine_ml.pkl')\n```\n\n### 2.3 Add Deduplication Before Signal Generation\n\n```python\n# Inside your signal generation loop\nfor ticker in candidate_tickers:\n    # Calculate signal metrics (confidence, pattern, etc.)\n    signal_data = calculate_signal(ticker)\n    \n    # CHECK 1: Deduplication (prevents META 9:59 + 10:29 duplicates)\n    should_fire, reason = analytics.should_fire_signal(ticker)\n    if not should_fire:\n        logging.info(f"⏸️ {ticker} blocked: {reason}")\n        continue\n    \n    # CHECK 2: ML Quality Prediction (optional confidence boost/penalty)\n    win_prob, confidence_adj = ml_loop.predict_signal_quality({\n        'rvol': signal_data['rvol'],\n        'vix': signal_data['vix'],\n        'score': signal_data['score'],\n        'time_of_day': datetime.now().strftime('%H:%M'),\n        'confidence': signal_data['confidence'],\n        'regime': signal_data['regime']\n    })\n    \n    # Adjust confidence based on ML\n    signal_data['confidence'] *= confidence_adj\n    \n    # Log signal to database\n    signal_id = analytics.log_signal(signal_data)\n    \n    if signal_id:\n        # Send Discord alert (only if signal_id is valid)\n        send_discord_alert(signal_data)\n```\n\n### 2.4 Monitor Active Signals (Every Minute)\n\n```python\n# In your main scanner loop (runs every 60 seconds)\ndef monitor_loop():\n    while market_open():\n        # Fetch current prices for active tickers\n        active_tickers = list(analytics.active_signals.keys())\n        current_prices = get_current_prices(active_tickers)\n        \n        # Check for T1/T2/Stop hits\n        analytics.monitor_active_signals(current_prices)\n        \n        # Your normal scanning logic\n        scan_for_new_signals()\n        \n        time.sleep(60)\n```\n\n### 2.5 End-of-Day Routines\n\n```python\n# At 4:00 PM market close\ndef market_close_routine():\n    logging.info("🔔 Market closed - Running EOD routines...")\n    \n    # 1. Generate and send daily report\n    reporter.run_eod_report()\n    \n    # 2. Retrain ML model on today's outcomes\n    ml_loop.retrain_daily()\n    \n    # 3. Reset deduplication for tomorrow\n    analytics.reset_daily_cooldowns()\n    \n    logging.info("✅ EOD routines complete")\n```\n\n---\n\n## Step 3: Signal Data Format\n\n### Required Fields for `log_signal()`\n\n```python\nsignal_data = {\n    'ticker': 'NVDA',\n    'signal_time': datetime.now(),\n    'pattern': 'EXPLOSIVE_BREAKOUT',  # or 'TRENDING_MOMENTUM', etc.\n    'confidence': 60.0,\n    'entry_price': 181.61,\n    'stop_loss': 180.16,\n    'target_1': 185.24,\n    'target_2': 187.42,\n    'regime': 'EXPLOSIVE',  # or 'TRENDING', 'REVERSAL'\n    'vix_level': 22.4,\n    'spy_trend': 'BULLISH',\n    'rvol': 4.0,\n    'score': 87.5,\n    'explosive_override': True  # Optional, default False\n}\n```\n\n---\n\n## Step 4: Environment Variables\n\nAdd to your `.env` or Railway environment:\n\n```bash\nDISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/YOUR_WEBHOOK_ID\n```\n\n---\n\n## What This System Does\n\n### ✅ **Deduplication (30-Minute Cooldown)**\n- **Problem:** META fired at 9:59 AM + 10:29 AM (duplicate signal)\n- **Solution:** `should_fire_signal()` blocks re-signals within 30 minutes\n- **Result:** Each ticker only fires once per setup\n\n### 🎯 **Auto-Tracking T1/T2/Stop**\n- **Problem:** Manual tracking of every trade outcome\n- **Solution:** `monitor_active_signals()` checks prices every minute\n- **Result:** Automatic detection of T1 hits, T2 hits, stop-outs\n\n### 🧠 **ML Learning Loop**\n- **Problem:** System doesn't learn from mistakes\n- **Solution:** ML model trained nightly on all outcomes\n- **Result:** Confidence boost (+10%) for high-probability setups\n\n### 📊 **Daily Reports**\n- **Problem:** No performance visibility\n- **Solution:** EOD report with W/L, P&L, pattern breakdown\n- **Result:** Discord notification at 4 PM with full stats\n\n---\n\n## Example Flow\n\n### **9:40 AM -
