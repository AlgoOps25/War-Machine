# Signal Analytics Integration Guide\n\n## Overview\n\nThis guide shows how to integrate the signal analytics system into your War Machine main loop.\n\n## Files Created\n\n1. **`src/analytics/signal_analytics.py`** - Core tracking + deduplication\n2. **`src/learning/ml_feedback_loop.py`** - ML training pipeline\n3. **`src/reporting/performance_reporter.py`** - Daily reports\n4. **`database/signal_outcomes_schema.sql`** - PostgreSQL schema\n\n---\n\n## Step 1: Create Database Tables\n\nRun this on your Railway PostgreSQL:\n\n```bash\npsql $DATABASE_URL -f database/signal_outcomes_schema.sql\n```\n\n✅ **This will:**\n- Create 3 tables: `signal_outcomes`, `pattern_performance`, `ml_training_data`\n- Add indexes for fast queries\n- Bootstrap with your NVDA 60% winner from today\n\n---\n\n## Step 2: Update `main.py` Imports\n\nAdd these imports at the top of your main script:\n\n```python\nfrom src.analytics.signal_analytics import SignalAnalytics\nfrom src.learning.ml_feedback_loop import MLFeedbackLoop\nfrom src.reporting.performance_reporter import PerformanceReporter\n```\n\n---\n\n## Step 3: Initialize After Database Connection\n\nAfter connecting to PostgreSQL, initialize the modules:\n\n```python\n# After: db = psycopg2.connect(DATABASE_URL)\n\n# Initialize analytics\nanalytics = SignalAnalytics(db)\nml_loop = MLFeedbackLoop(db)\nreporter = PerformanceReporter(db, discord_webhook_url=DISCORD_WEBHOOK)\n\nlogging.info(\"[MAIN] Analytics system initialized\")\n```\n\n---\n\n## Step 4: Add Deduplication Check Before Discord Alert\n\n**CRITICAL:** This prevents duplicate signals (META/QQQ firing twice)\n\n```python\n# BEFORE sending Discord alert, add this check:\n\nfor signal in new_signals:\n    ticker = signal['ticker']\n    \n    # ✅ DEDUPLICATION CHECK\n    should_fire, reason = analytics.should_fire_signal(ticker)\n    if not should_fire:\n        logging.info(f\"⏸️ {ticker} blocked: {reason}\")\n        continue  # Skip this signal\n    \n    # ✅ LOG TO DATABASE\n    signal_data = {\n        'ticker': ticker,\n        'signal_time': datetime.now(),\n        'pattern': signal['pattern'],\n        'confidence': signal['confidence'],\n        'entry_price': signal['entry'],\n        'stop_loss': signal['stop'],\n        'target_1': signal['t1'],\n        'target_2': signal['t2'],\n        'regime': get_current_regime(),\n        'vix_level': get_vix(),\n        'spy_trend': get_spy_trend(),\n        'rvol': signal['rvol'],\n        'score': signal['score'],\n        'explosive_override': signal.get('explosive_override', False)\n    }\n    \n    # ✅ ML CONFIDENCE ADJUSTMENT (optional)\n    confidence_adj, win_prob = ml_loop.predict_signal_quality(signal_data)\n    signal_data['confidence'] += confidence_adj\n    \n    signal_id = analytics.log_signal(signal_data)\n    \n    # ✅ SEND DISCORD ALERT (only if signal_id exists)\n    if signal_id:\n        send_discord_alert(signal, confidence_boost=confidence_adj)\n```\n\n---\n\n## Step 5: Monitor Active Signals Every Minute\n\nIn your scanner loop (runs every 60 seconds):\n\n```python\n# In your main loop (every 1 minute)\n\nwhile market_is_open():\n    # ... your existing scanner logic ...\n    \n    # ✅ MONITOR ACTIVE SIGNALS\n    current_prices = {}\n    for ticker in analytics.active_signals.keys():\n        current_prices[ticker] = get_current_price(ticker)\n    \n    analytics.monitor_active_signals(current_prices)\n    \n    time.sleep(60)  # Wait 1 minute\n```\n\n---\n\n## Step 6: Add Market Open/Close Routines\n\n### At Market Open (9:30 AM):\n\n```python\nif datetime.now().time() == time(9, 30):\n    analytics.reset_daily_cooldowns()\n    logging.info(\"[MAIN] 🔄 Daily reset complete\")\n```\n\n### At Market Close (4:00 PM):\n\n```python\nif datetime.now().time() == time(16, 0):\n    #
