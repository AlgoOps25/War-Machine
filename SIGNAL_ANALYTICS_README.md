# 📊 Signal Analytics System

## Overview

The Signal Analytics system tracks every signal generated by War Machine and monitors their outcomes to calculate win rates, profit factors, and other performance metrics. This enables data-driven tuning of the signal generator logic.

## Features

✅ **Automatic Signal Logging** - Every signal is logged to PostgreSQL with full details  
✅ **Outcome Tracking** - Monitors when signals hit target, stop loss, or expire  
✅ **Performance Metrics** - Win rate, profit factor, average R:R, hold duration  
✅ **Breakdown Analysis** - Stats by ticker, direction, confidence level  
✅ **Historical Database** - All signals stored for backtesting and analysis  

## Database Schema

### signals Table

```sql
CREATE TABLE signals (
    id              SERIAL PRIMARY KEY,
    ticker          TEXT NOT NULL,
    direction       TEXT NOT NULL,
    signal_time     TIMESTAMP NOT NULL,
    entry_price     REAL NOT NULL,
    stop_price      REAL NOT NULL,
    target_price    REAL NOT NULL,
    risk            REAL NOT NULL,
    reward          REAL NOT NULL,
    risk_reward     REAL NOT NULL,
    confidence      INTEGER NOT NULL,
    volume_multiple REAL,
    atr             REAL,
    signal_type     TEXT,
    
    -- Outcome tracking (updated when signal closes)
    outcome         TEXT,              -- 'WIN', 'LOSS', 'EXPIRED'
    exit_price      REAL,
    exit_time       TIMESTAMP,
    pnl             REAL,              -- Dollar P&L
    pnl_r           REAL,              -- P&L in R (risk units)
    hold_duration   INTEGER,           -- Minutes held
    
    notes           TEXT,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## Performance Metrics Explained

### Win Rate
```
Win Rate = (Wins / Total Completed Signals) × 100
```
- **Target:** 55-65% for breakout systems
- **Below 50%:** Tighten entry criteria
- **Above 70%:** May be too conservative

### Profit Factor
```
Profit Factor = Total Wins $ / Total Losses $
```
- **Target:** 2.0+ for profitable systems
- **Below 1.5:** Adjust R:R or tighten stops
- **Above 3.0:** Excellent edge

### Average R (R-Multiple)
```
Average R = (Total P&L in R) / Total Signals
```
- **Positive:** Profitable on average
- **Target:** 0.5R+ per signal
- **1R+:** Very strong system

### Average Hold Duration
- **< 30 mins:** Scalping system
- **30-120 mins:** Intraday swing
- **> 2 hours:** Position holding

## Signal Flow

```
1. Scanner detects breakout
   ↓
2. signal_generator.check_ticker()
   ↓
3. log_signal() → Stores in DB with ID
   ↓
4. Signal fires → Discord alert
   ↓
5. Scanner monitors price
   ↓
6. Target/Stop hit OR Expired
   ↓
7. update_signal_outcome() → Updates DB
   ↓
8. Performance stats updated
```

## Tuning Based on Stats

### If Win Rate < 50%

**Option 1:** Increase `min_confidence`
```python
# In signal_generator initialization
signal_generator = SignalGenerator(
    min_confidence=70  # Up from 60
)
```

**Option 2:** Increase `volume_multiplier`
```python
signal_generator = SignalGenerator(
    volume_multiplier=2.5  # Up from 2.0
)
```

**Option 3:** Tighten ATR stop
```python
detector = BreakoutDetector(
    atr_stop_multiplier=1.2  # Down from 1.5
)
```

### If Profit Factor < 1.5

**Option 1:** Widen targets
```python
detector = BreakoutDetector(
    risk_reward_ratio=2.5  # Up from 2.0
)
```

**Option 2:** Tighter stops
```python
detector = BreakoutDetector(
    atr_stop_multiplier=1.2  # Down from 1.5
)
```

### If Average Hold > 4 Hours

**Option 1:** Reduce target size
```python
detector = BreakoutDetector(
    risk_reward_ratio=1.5  # Down from 2.0
)
```

**Option 2:** Use time-based exits
```python
# In signal_generator
signal_generator.clear_expired_signals(
    max_age_hours=2  # Down from 4
)
```

## By-Ticker Analysis

Check which tickers are most profitable:

```python
stats = analytics.get_performance_stats(days=30)
for ticker, data in stats['by_ticker'].items():
    total = data['wins'] + data['losses']
    wr = (data['wins'] / total * 100) if total > 0 else 0
    print(f"{ticker}: {wr:.0f}% WR | ${data['pnl']:+.2f} P&L")
```

## Database Queries

### Recent Winners
```sql
SELECT ticker, entry_price, target_price, pnl, pnl_r, hold_duration
FROM signals
WHERE outcome = 'WIN'
ORDER BY pnl DESC
LIMIT 10;
```

### Worst Performers
```sql
SELECT ticker, entry_price, stop_price, pnl, confidence
FROM signals
WHERE outcome = 'LOSS'
ORDER BY pnl ASC
LIMIT 10;
```

### Win Rate by Confidence
```sql
SELECT 
    CASE 
        WHEN confidence >= 80 THEN '80+'
        WHEN confidence >= 70 THEN '70-79'
        WHEN confidence >= 60 THEN '60-69'
        ELSE '<60'
    END as conf_bucket,
    COUNT(*) as total,
    SUM(CASE WHEN outcome = 'WIN' THEN 1 ELSE 0 END) as wins,
    AVG(pnl_r) as avg_r
FROM signals
WHERE outcome IN ('WIN', 'LOSS')
GROUP BY conf_bucket
ORDER BY conf_bucket DESC;
```

## Integration with Scanner

The analytics system is automatically integrated:

1. **Signal Generator** calls `log_signal()` when signal fires
2. **Scanner** monitors active signals every cycle
3. **Outcome tracking** calls `update_signal_outcome()` on target/stop hit
4. **Daily cleanup** marks stale signals as EXPIRED

No manual intervention required - just run the scanner and stats accumulate!

## Backup and Export

### Export to CSV
```python
import csv
from signal_analytics import analytics

signals = analytics.get_recent_signals(1000)
with open('signals_export.csv', 'w', newline='') as f:
    writer = csv.DictWriter(f, fieldnames=signals[0].keys())
    writer.writeheader()
    writer.writerows(signals)
```

### Backup Database
```bash
# PostgreSQL
pg_dump -h localhost -U postgres -d war_machine -t signals > signals_backup.sql

# Restore
psql -h localhost -U postgres -d war_machine < signals_backup.sql
```
